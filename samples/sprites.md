# mod_sprites

レトロゲームやレトロコンソールのキャラクターを画面上に散らすモジュールです

- キャラクターデータはスプライトで定義します
- スプライトの定義データですが、1スプライト図形の中に使える色は16色、サイズは64x64の制限を設けています
- また、スプライトの行数は1ドット単位ですが、横幅は8ドット単位になります

## wallpaper本体のパラメータ

- 背景色、背景色2： spriteモジュールの背景は、開始色=背景色、終了色=背景色2の斜めグラデーションとなります
- 彩度： スプライトがフルカラーのため、壁紙に落ち着いた色を使いたい場合は彩度を調整してください
- 星：背景に散らす星の密度になります。0で星無しになります。背景に星が流れる系のキャラクター配置時に設定すると吉
- パターン拡大：1ドットを何ピクセル四方にするかを決定します。1だと何が何だか。
- 間隔：スプライトを配置する時のスプライト間の間隔です
- 密度： あまり明には影響しませんが、スプライトの配置密度の係数です

## スプライトセットエディタ

プレビュー画像をクリックすると、スプライトセットエディタ画面になります

- セットエディタでは、スプライトセットの読み込み(Load)、書き出し(Save)、イメージ出力(Dump)の他、表示アイテムの選択(Pick Item)、新規取込(Import)、最後に追加したスプライトの削除(Purge) が行えます
- Load: プルダウンで読み込むスプライトセットを選択し、Loadボタンで実行。読み込むsprrファイルはsamplesフォルダに配置されている必要があります
- Save: Saveボタンを押下すると出力先ファイル名の入力になります。ディレクトリを変更できます
- Dump: Dumpボタンで画像イメージでスプライトを出力します。出力先ディレクトリは samples / セット名 です
- Pick Items: セット内のスプライトのどれを表示するか選択できます。選択されなかったスプライトはSaveでも出力されません
- Import: スプライトを画像ファイルから取り込みます(後述)
- Purge: 最後に追加したスプライトを削除します。複数削除や、末尾以外を削除したい場合はsprファイルをエディタで編集してください

#### アイテム選択画面

- チェックボックスで単品選択/解除を行うほか、Selecte all、Clear all ボタンで全選択/全解除ができます

#### インポート画面

- Read File: 画像ファイルの読み込みを行います。
  - PNG,JPG,GIF,ICOがデフォルト対象になっていますが、PILのImage.openで読み込めるファイル形式ならどれでも可
  - 画像ファイルのサイズが64x64よりも大きかった場合、**64x64に縮小されます**
  - スプライトの幅は8の倍数ドットになります
  - 画像ファイルの色数が16色以上の場合、色数を16色に制限します
  -  **sprファイルを直接編集する場合は全体16色制限はありません**
- Reduce Color: 上部のColor/Rowボックスの数値で、1ライン毎の色数を減色します。**ライン当たりの色数は透過色を含め16色までです**
- Transparent: 読み込んだファイルで利用されている色のパレットをクリックすると、その色が透過色に設定されます(枠が付きます)
- ID: 読み込んだファイル名がデフォルトで入りますが、入力ボックスのテキストを変更して登録前に修正することもできます
- Register: ID欄の名前でスプライトをスプライトセットに追加します
- BulkRead: 指定フォルダ下の画像ファイルを一括でスプライトセットに変換します。透過色はセット共通で1色指定になります。微調整はセット読み込みの後、ReadFileやテキストエディタで。

#### 新規のセットを作成したい

- 方法① BulkReadで読み込んだ後、DATA_DIR(デフォルトはsample)の下にsaveすることで新規スプライトセットにできます。
- 方法② スプライトセットエディタ画面で、内蔵パターンを全部purgeしてから、importで必要な数のスプライトを登録後、方法①同様にsaveしてください
- 方法③ 完全に新規作成する場合は、テキストエディタなどでsprファイルの仕様に沿って作成するのもありです(大変ですが)

#### sprファイルのフォーマット

- 先頭が#の行、および空行は読み飛ばします。最初の#行の内容がスプライトセットの説明としてセットエディタで表示されるので、(C)などを記載するとよいでしょう
- スプライトファイルには、複数のスプライトパターンを含むことができます。ただし、スプライト名はファイル内ユニークになっている必要があります
- 個々のスプライトは独立なので、スプライトをまたがった色数制限などはありません
- 1つのスプライトパターンは、1行目に [スプライト名] を書き、2行目は 幅,高さ を規定します。幅、高さは数値で、0xをつけて16進で記述しても構いません
- スプライトの3行目以降は実際のパターン定義で、1行で1ラインを表します。
- やや面倒なのですが、使用する色数だけ ビットパターンを数値で並べた後に、色指定文字列を並べます。区切り文字はカンマです
- 例： 無赤赤青赤無青黄 → 0x68, 0x12,0x01, #ff0000, #0000ff, #ffff00
- マクロコマンドとして、第一要素無し、第二要素にコマンドを置くことができます
  - ,turnover : コマンド行の前までの行を逆順に追加します。スプライト高さに満たなければ続けてパターン行を記述できます、中心で上下折り返しになるパターン用
  - ,rep# : コマンド行の前のラインを#ライン繰り返します
