# mod_memphis  メンフィス風パターン

メンフィス（Memphis）は1981年に結成された多国籍からなるデザイナー集団で、1980年代前半にイタリアを中心に世界のデザイン・建築に影響を及ぼした。またデザインにおけるポストモダンの代表としても一世を風靡した。 

- 幾何学模様を散らした感じのデザインをメンフィスパターンと呼ぶそうです
- それっぽいパターンを生成することを目指して、小さな図形描画関数と、それを組み合わせて背景上に散布するプログラムを組み合わせています

## wallpaper本体での設定

- 背景色：縦グラデーションの開始色・終了色を指定します
- 彩度： 画像全体の彩度を指定します。「壁紙がビビッドなのは…」という向きには彩度を抑えてご利用ください
- パターンサイズ： 最大サイズの指定になります
- トライ数：パターン配置数と言いたいところなのですが、衝突回避などで狙った数が置けない場合があり、試行回数としています
- 間隔：パターン同士の間隔です

## 拡張設定画面

- プレビュー画面をクリックすると拡張設定画面が開きます
- 拡張設定画面では、チェックボックスにチェックを入れることで表示する図形セットの選択が可能です
- 基本図形に加え拡張図形も選択可能です

## パターンの追加

パターン生成関数は以下のテンプレートで作成してください

```
@register
def pattern_script(size: int, color: tuple):
	image = Image.new('RGBA', (size,size), 0)
	<生成コード>
	：
	return image
```

'RGBA'モード の PILイメージを返す必要があります。

また、この関数を拡張設定で選択可能にするため、グローバル変数に関数名を追加します

```
APPENDS = [
    ('circle_solid', 'circle_lattice'),
    ('square_solid', 'square_bias'),
	：
    ('burger', None),
    ]
```

APPENDSに関数名 (**関数ではなく、関数名文字列です**) を登録するのですが、メンフィスデザインではよくソリッドな図形と輪郭の重ね描きもみられることから、最大２つの要素をセットで登録するようになっています。作成した図形が重ね書き不要の場合は、タプルの第二要素にNoneを指定してください。

APPENDSの前に定義されているSHAPESは、基本図形の定義になります。個々の表示選択ではなく、SHAPESに含まれる全セットを一括でON/OFFする事になるので、ユーザ追加図形についてはAPPENDSに追加することを推奨します
